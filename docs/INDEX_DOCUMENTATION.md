## 📋 Cas d'usage

### Cas 1: Nouveau développeur rejoint le projet
```
1. Lire README_BTP_OPTIMISATIONS.md
2. Installer le projet (COMMANDES_UTILES.md)
3. Lancer en local (npm start)
4. Explorer le code (frontend/src/pages/BTP.jsx)
5. Lire RESUME_FINAL_OPTIMISATIONS.md
```

### Cas 2: Bug en production
```
1. Identifier le problème
2. Consulter GUIDE_DEPLOIEMENT_OPTIMISE.md > Troubleshooting
3. Tester en local (COMMANDES_UTILES.md)
4. Corriger le bug
5. Tester (TESTS_VALIDATION_BTP.md)
6. Déployer (GUIDE_DEPLOIEMENT_OPTIMISE.md)
```

### Cas 3: Nouvelle fonctionnalité
```
1. Planifier la fonctionnalité
2. Développer en local
3. Tester (TESTS_VALIDATION_BTP.md)
4. Vérifier Lighthouse (COMMANDES_UTILES.md)
5. Commit et push (COMMANDES_UTILES.md > Git)
6. Déployer (GUIDE_DEPLOIEMENT_OPTIMISE.md)
```

### Cas 4: Audit de performance
```
1. Lire RAPPORT_ANALYSE_PAGE_BTP.md
2. Exécuter les tests (TESTS_VALIDATION_BTP.md)
3. Analyser avec Lighthouse (COMMANDES_UTILES.md)
4. Comparer avec RESUME_FINAL_OPTIMISATIONS.md
5. Identifier les régressions
6. Corriger et re-tester
```

### Cas 5: Mise à jour des dépendances
```
1. Vérifier les dépendances (COMMANDES_UTILES.md > Maintenance)
2. Mettre à jour (npm update)
3. Tester localement (TESTS_VALIDATION_BTP.md)
4. Vérifier Lighthouse
5. Déployer en preview (vercel)
6. Valider et déployer en prod
```
